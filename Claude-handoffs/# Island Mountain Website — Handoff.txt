3. **Before committing, ALWAYS run this verification:**
```bash
for f in *.html blog/*.html; do
  last=$(tail -1 "$f" | tr -d '[:space:]')
  if [ "$last" != "</html>" ] && [ "$f" != "googlecff518dc414acaa3.html" ]; then

  fi
done
```

### Navigation Structure
- **Desktop:** Horizontal nav with "Solutions" dropdown containing 5 vertical pages (Law Firms, Medical Practices, Tribal Nations, Research Labs, Defense Contractors)
- **Mobile:** Hamburger menu opens sidebar with accordion for Solutions sub-links
- **Active link highlighting:** JS matches current filename to nav hrefs, adds `.active` class
- **Solutions dropdown parent:** Gets `.active` class when any child page is active

### CSS Architecture (style.css)
- Single file, no preprocessor
- Responsive breakpoints throughout (media queries)
- **Scroll-reveal animation (PROGRESSIVE ENHANCEMENT):**
  - `.fade-in` — base class, has transition properties only
  - `.fade-in.fade-hidden` — JS-added class that hides elements (opacity:0, translateY:30px)
  - `.fade-in.visible` — JS-added class that reveals elements (opacity:1, translateY:0)
  - **If JS fails, content is visible by default** — `.fade-in` alone has no opacity:0

### JavaScript Architecture (main.js)
- IIFE wrapper, `'use strict'`
- **Navbar scroll effect:** Adds `.scrolled` class when scrollY > 50
- **Mobile menu:** Hamburger toggle, overlay click-to-close, link click-to-close
- **Solutions accordion:** Mobile-only toggle for Solutions sub-links
- **Active nav highlighting:** Matches `window.location.pathname` to link hrefs
- **IntersectionObserver (scroll reveal):**
  1. JS adds `.fade-hidden` to all `.fade-in` elements (hides them)
  2. Observer triggers `.visible` class when element enters viewport (threshold: 0.1)
  3. **Safety net:** setTimeout at 2 seconds forces `.visible` on any still-hidden elements
  4. Falls back gracefully if IntersectionObserver not supported (content stays visible)
- **Particle canvas:** Full-page star field on `#particles-canvas`, drifting dots with twinkle animation, color shifts from cool blue to warm amber as user scrolls
- **Smooth scroll:** Anchor links use `scrollIntoView({ behavior: 'smooth' })`

### FAQ Page (faq.html) — Special Structure
- 25 FAQ items using `.faq-item` / `.faq-item.active` toggle pattern
- **Inline `<script>` at bottom of faq.html** (NOT in main.js) handles accordion:
  - Click `h3` inside `.faq-item` toggles `.active` class
  - Only one FAQ open at a time (closes others on click)
- **AEO optimization:** First ~40 words of each answer wrapped in `<strong>` tags for AI answer engine extraction
