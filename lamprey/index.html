Data, Inputs, Behavior

Router classification

Inputs: public text, email, SSN, PHI terms, classified markers, upstream flags.

Behavior: highest sensitivity wins; critical denies; regulated content stays local; public queries can route to cloud.

Entity detection

Inputs: medical terms, tribal sovereignty terms, ITAR/EAR phrases, unrelated text.

Behavior: raw matched text is never emitted; only hashed spans and entity kinds are available for audit/routing.

Policy modules

Inputs: sensitivity class, profile role, token estimate, medical/tribal/export entities, upstream flags, and module TOMLs.

Behavior: highest-priority matching rule wins, ties resolve by restrictiveness, deny/reroute can override default routing, and allow/flag fall through to default decisions.

Compliance reports

Inputs: audit log records, module filters, routing decisions, report type, output format, and requested time window.

Behavior: inverted date ranges are rejected, generated reports round-trip through the API, and list/query responses expose report metadata without leaking raw subject identifiers.

Trust gate

Inputs: tenant, subject, claim ID, scopes, revocation status, local-dev token exchange, and trust snapshots.

Behavior: revoked or missing trust blocks protected paths, trust status reports air-gapped defaults, and local-dev exchanges produce scoped test tokens for integration workflows.

HIPAA compliance

Inputs: SSNs, emails, phone numbers, URLs, IPs, MRNs, dates, ZIPs, VINs, clinical names, medications, ICD-10 codes, lab values, and BAA modes.

Behavior: PHI is hashed or redacted before audit/cloud paths, BAA rules force local when required, and PHI scanning stays under the latency budget.

Cost controls

Inputs: estimated tokens, profile role, current usage, zero-token requests, cloud failures.

Behavior: hard caps force local or block, soft caps warn at 80%, failed cloud calls do not burn budget.

Scheduler + KV

Inputs: topology fixtures, queue depth, VRAM pressure, priority, KV candidates, batch membership.

Behavior: placement scores explain latency/memory/topology/eviction/batch factors; system priority and warm continuations are protected.

API + auth

Inputs: API keys, profile roles, missing/invalid tokens, rate-limit bursts, health exceptions.

Behavior: strict auth blocks spoofing, health remains exempt, MAI error codes and retry headers are stable.

Trace privacy

Inputs: NDJSON trace events, session IDs, prompt-like fields, replay policies.

Behavior: allowlisted fields only, salted session hashes, deterministic replay comparisons, and simulator policy reports pass on Windows.
